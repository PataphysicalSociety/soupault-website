<h1 id="post-title">Migrating from soupault.app to soupault.net</h1>

<p>Date: <time id="post-date">2026-01-24</time> </p>

<p id="post-excerpt">
The website has migrated to a new domain: <a href="https://soupault.net">soupault.net</a>.
I plan to eventually let the old soupault.app domain lapse, so please update your links!
</p>

When I started Soupault in 2019 I registered `soupault.app` domain for it.
There was no particular rational reason: Soupault was an application, the `.app` TLD was fashionable
and looked fitting, so I chose it.

Later I realized that there were more reasons not to use that domain than reasons to use it.
First, it's owned by Google, and there are hardly reasons to voluntarily fund a company
that wants to monopolize the World Wide Web.

Second, that domain acquired a pretty bad reputation due to its use for phishing —
`.app` is a common file extension for macOS applications, so it came to be used
to trick people into opening links without realizing that what they thought was a file extension
is in fact a TLD.

That created a significant backlash against `.app`, `.mov`, and `.zip`; and many security scanners
are biased against those TLDs — I'd say, rightfully so.

So, I registered `soupault.net` and set up a 301 redirect from the old domain to the new one.

I plan to keep `soupault.app` active for about a year to prevent link rot,
but if you have links to `soupault.app` on your websites, please update them with the new domain
when you have a chance.
